## 1. 市場分析

### 競合マッピング

| アプリ | 価格 | プラットフォーム | 主な機能 | 欠点 |
|---|---|---|---|---|
| ChemDraw Direct | $350/年 | クラウド | 本格的な構造描画と命名 | 高額、研究者向け |
| MolView | 無料 | Web | 3D 可視化 | ΔH 計算なし |

**ポジショニング**: 「ChemDraw と電卓の中間」、**学生・教員・プロセス初期設計向けの軽量計算機**。

初年度リーチ可能ユーザー数 (楽観): 500 万 × 0.6 × 0.02 = **~6 万人**

## 2. 価格モデル選択肢

### A. 完全無料 (ads / donations)

- DL 最大化、レビュー数増
- Microsoft Advertising SDK は終了済み → 外部広告モジュール必要 (技術的に面倒)
- 推奨度: **低** (収益化しない場合のみ)

### B. 買い切り $2.99

- **月 100 DL × $2.99 × 85% = $254/月 ≈ 38,000 円/月**
- シンプル、返金リスク小
- 学生に少し高く感じる価格帯

### C. 買い切り $0.99 (アンカー戦略)

- **月 300 DL × $0.99 × 85% = $253/月** ← B と同じ収益を DL 3倍で
- 衝動買いされやすい
- 長期 CLV 伸びにくい

### D. フリーミアム (推奨)

| ティア | 価格 | 含まれる機能 |
|---|---|---|
| Free | $0 | 結合エネルギー法、Benson 群加算法、英日 UI、オフライン |
| Pro (買い切り) | $4.99 | 計算履歴、CSV/PDF エクスポート、PubChem 連携、バッチ計算、プレミアムサポート |

- 無料 DL 最大化 → Pro 転換率 3〜5% を想定
- **月 1,000 DL × 4% × $4.99 × 85% = $170/月** (開始直後)
- 成熟後は累計無料 DL 10,000 → **月 $340 ペース**

### E. サブスク $1.99/月

- LTV 最大
- 継続解約のリテンション管理が重要
- **クライアント完結アプリではサブスクの価値訴求が難しい** (毎月何が新しいのか)
- 推奨度: **低** (サーバー処理を提供する Pro+ 層を作ってから)

### F. ハイブリッド (一次推奨)

1. **v1.0**: 完全無料で公開 → 初期 DL を稼ぐ + レビュー獲得
2. **v1.5 (3ヶ月後)**: Pro 機能を追加し D モデルへ移行
3. **v2.0 (1年後)**: バックエンド計算を追加し Pro+ サブスクを導入

最も安全・段階的・学びながら進められる。

## 3. 初期推奨: ハイブリッド F モデル

### フェーズ 1 (v1.0.0): 完全無料で公開

理由:
- レビュー・評価の獲得が最優先
- 競合不在の領域なので価格差別化より認知獲得
- ユーザーインタビュー的に機能要望を集める期間とする

### フェーズ 2 (v1.5.0): Pro 機能リリース

v1.5.0 までに実装したい Pro 限定機能:

1. **計算履歴** (localStorage に保存、CSV エクスポート)
2. **PDF レポート出力** (反応式 + 内訳を A4 印刷)
3. **分子式一括入力** (テキスト貼付で複数反応を一括計算)
4. **反応テンプレート** (よく使う反応の保存・呼出)
5. **ダークモード**
6. **Benson パラメータのカスタマイズ** (高度ユーザー向け)

価格: **$4.99 買い切り (Pro)** + **無料版は現状機能維持**

### フェーズ 3 (v2.0.0): Pro+ サーバー計算

`backend/` の xTB + PySCF 連携を生かす:
- 分子最適化 (xTB)
- 反応座標サーチ (遷移状態)
- ab initio レベルの ΔH 精密化

サーバーコスト発生 → **サブスク $4.99/月 または従量課金**が合う

## 4. Microsoft Store での IAP 実装メモ

### In-App Purchase タイプ

Store は 2 種類の IAP をサポート:

| タイプ | 特徴 | 本アプリでの用途 |
|---|---|---|
| **Durable** | 買い切り、永続有効 | Pro アンロック ← これを使う |
| **Consumable** | 使い捨て (例: ゲームコイン) | 本アプリでは不要 |
| **Subscription** | 定期課金 | Pro+ で将来使う |

### 技術的実装 (フェーズ 2 で対応)

```js
// Store IAP は WinRT API 経由。PWA から直接は呼べない。
// PWABuilder で MSIX 化する際に Windows.Services.Store を wrapper 経由で利用可能。

// 参考: https://learn.microsoft.com/en-us/windows/apps/publish/in-app-purchases
```

PWA では外部決済リンク (Stripe 等) を使い Store 手数料 15% を回避する手もあるが、
Microsoft Store の規約変更 (2024年〜) により外部決済も許可されている。
ただし **Microsoft による顧客サポート**の対象外になるため、初期は Store 内 IAP 推奨。

## 6. リスク & 対策

| リスク | 影響 | 対策 |
|---|---|---|
| Store 審査落ち | 公開遅延 | PWABuilder スコア 90+ で事前検証 |

| 化学パラメータ不足への要望 | 機能開発圧力 | OSS で PR 受付にする、または Pro ユーザー投票 |
